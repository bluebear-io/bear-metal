# DEN-2332 — Backend DB: Postgres alongside SQLite via DATABASE_URL

> Linear: https://linear.app/bluebearsecurity/issue/DEN-2332

Make the dashboard backend speak Postgres in prod while keeping SQLite as the default for local dev / tests. Mirror the manager's `src/manager/tasks.ts` URL-scheme factory pattern, but keep Drizzle (the dashboard's ORM) rather than going raw SQL.

## Constraints

- `BEAR_METAL_DB_PATH` → `BEAR_METAL_DATABASE_URL` (`sqlite:./path.db` | `postgres://...`).
- Two parallel Drizzle schemas (`sqliteTable` + `pgTable`) producing **identical** inferred TS row types. The rest of the codebase keeps referencing one `Ticket`/`NewTicket`/etc.
- All writer/repository signatures become **async** (Drizzle PG returns Promises; SQLite paths still work via `await` of plain values).
- Schema setup at startup is dialect-aware raw `CREATE TABLE IF NOT EXISTS` — same pattern the manager already uses.
- No changes in `src/manager/` or `src/worker/`.

## Tasks (low-risk first)

### T1 — Config: rename to `BEAR_METAL_DATABASE_URL`
`src/backend/config.ts` + `src/backend/config.test.ts`. Required env, defaults removed (caller decides). Update `.env.example`.

### T2 — Parallel `pgTable` schema
`src/backend/db/schema.ts` keeps the existing `sqliteTable` exports unchanged. New `src/backend/db/schema-pg.ts` defines the same six tables with `pgTable` — `text` everywhere for strings (no `pgEnum` — matches SQLite's lack of enforcement and keeps `Ticket["bmStatus"]` shape stable), `timestamp({ withTimezone: true })` for the time columns, `boolean` for `draft`/`merged`. Both expose `enumValues` constants the ingest validator reads. Add a `schema-test.ts` (vitest) that asserts the two `$inferSelect` types are TypeScript-equivalent via a runtime row round-trip on each.

### T3 — Bootstrap SQL per dialect, moved into `client.ts`
Move the `SCHEMA_SQL` const out of `index.ts` and into `src/backend/db/client.ts`. Add a `SCHEMA_SQL_PG` next to it. Expose `applySchema(databaseUrl, sqlite | pool)` that runs the right one on first connect (idempotent).

### T4 — Connection factory
`src/backend/db/client.ts`: add `openWritableDbFromUrl(url)` and `openReadOnlyDbFromUrl(url)` that pick driver by URL scheme. Return `{ db, close }` with a union `Db = BetterSQLite3Database<typeof schemaSqlite> | NodePgDatabase<typeof schemaPg>`. Bootstrap (T3) runs inside the writable factory; the read-only factory just opens with `{ readonly: true }` for SQLite and a read-only `pg.Pool` for PG. Keep `foreign_keys = OFF` behavior for SQLite (best-effort writer); for PG, the equivalent is `SET session_replication_role = 'replica'` per pooled connection or accept that PG's FK enforcement is stricter and rely on retry logic — start with `session_replication_role` and document. Deprecate the path-taking variants (`openReadOnlyDb`/`openReadWriteDb`) — keep them as thin wrappers that build a `sqlite:<path>` URL, so the seeder + backfill + index continue to work with minimal churn until they're updated.

### T5 — Make writer functions async
`src/backend/db/writer.ts`: every `upsert*` / `insertEvent` becomes `async (db, p): Promise<void>` and `await`s the Drizzle call. Same for tests. Ingest route handlers (`src/backend/routes/ingest.ts`) become async and `await` the writer calls — `handle()` helper switches to `async (req, res)`.

### T6 — Repository remains read-only sync where it can, async at the boundary
`src/backend/db/repository.ts` builds tables via Drizzle — make every query method `async`, returning Promises. Update callers in `src/backend/routes/` accordingly.

### T7 — Async-ify backfill writer + entry
`src/backend/backfill/writer.ts` (`ensureBackfillWorker`, `writeBundle`) becomes async; `src/backend/backfill/index.ts` `runBackfill` already async — propagate `await`s.

### T8 — Async-ify mock seeder
`src/backend/mock/seed.ts`: `seedMockData` and `main` become async. CLI guard awaits.

### T9 — Wire `index.ts` to the factory
`src/backend/index.ts`: drop the inlined `SCHEMA_SQL`, call `openWritableDbFromUrl(config.databaseUrl)`, log the chosen dialect. Shutdown handler awaits `close()`.

### T10 — Postgres-path test
Add `pg-mem` as a devDependency (exact-pinned, quarantine-compliant). New `src/backend/backfill/writer.pg.test.ts` runs `writeBundle` against a `pg-mem`-backed Drizzle handle and asserts the same row counts as the SQLite test. One PG path proved in CI is enough — full PG coverage isn't the goal.

### T11 — README + `.env.example`
Add a "DB configuration" section to README with both URL schemes; update `.env.example`.
