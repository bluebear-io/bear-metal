# DEN-2565 — Bear Metal worker environment builder startup hook

## Goal
Add an operator-controlled startup hook (`WORKER_ENVIRONMENT_BUILDER_COMMAND` / `WORKER_ENVIRONMENT_BUILDER_PATH`) that runs **once** when the Bear Metal process starts, **before** `scheduler.start()` / `taskWorker.start()`. Use it to install toolchains (Go, Rust, Python, pnpm, …) that the per-ticket workspace builder may then need.

## Behavior contract
- Both env vars optional. Neither set → no-op, normal startup.
- Exactly one set → run it once at startup.
- Both set → fail startup with a clear config error.
- Non-zero exit → fail startup, do not start scheduler / task worker.
- Runs in the Bear Metal process environment (inherits `process.env`). It is **not** scoped to `AGENT_WORKDIR` and is not per-ticket.
- Not exposed to the coding agent (operator startup code only).

## Files to touch (bear-metal repo)
- `src/manager/config.ts` — add `workerEnvironmentBuilderCommand` / `workerEnvironmentBuilderPath` (both optional, mutually exclusive). Both null when unset.
- `src/manager/worker-env-builder.ts` *(new)* — `runWorkerEnvironmentBuilder({ command, path, logger })`. Wraps inline command into a temp `bash` script (same shebang/`set -euo pipefail` pattern as `clone.ts`), executes via `runCommand`, logs start/success/failure, cleans up temp dir.
- `src/manager/worker-env-builder.test.ts` *(new)* — covers no-op, command-only run, path-only run, both-set rejection, non-zero exit rejection.
- `src/manager/config.test.ts` — add cases: neither / command-only / path-only / both-set.
- `src/manager/index.ts` — invoke `runWorkerEnvironmentBuilder` after `loadConfig` + logger setup, **before** `scheduler.start()` / `taskWorker.start()`. Let throws propagate (process exits non-zero, which is the required behavior).
- `README.md` — add the two env vars to the table, add a “Worker environment builder” section near “Workspace builder”, contrast it with per-ticket workspace setup, include a Go toolchain example.
- `.env.example` — commented examples, note mutual exclusivity and optionality.

## Open decisions
- Timeout for the builder: match `clone.ts` workspace-builder timeout of 10 min — operators can re-run if a toolchain install needs longer; nothing in the spec dictates a different number.
- Wiring test for `index.ts` itself: skipped — `index.ts` runs at import time and has no existing unit tests. Behavior is covered by the new `worker-env-builder.test.ts` (which proves non-zero exit throws) plus code review of the call site (a single `await` line before `scheduler.start()` / `taskWorker.start()`).
