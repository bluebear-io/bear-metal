# DEN-2467 — Lazy-load tool call details in ticket view

## Problem
`GET /api/tickets/:id` returns all tool call data (args, result, thought) for every run in one response. On tickets with many tool calls this is large and slow. Detail fields should be fetched only when the user expands a row.

## Backend changes
- `src/db/client.ts`
  - Trim `RunToolCallRow` (returned by `getTicketDetail`) to `{ id, runId, sequence, toolName, resultStatus, createdAt }`. Strip `argsJson`, `resultText`, `outputSize`, `thoughtText`.
  - Add `ToolCallDetail` interface `{ argsJson, resultText, outputSize, thoughtText }`.
  - Add `getToolCallDetail(runId, sequence)` which reads `tool_calls_json` from `tasks` where `id = runId`, parses the JSON array, and returns the entry matching `sequence` (or null).
- `src/manager/routes.ts`
  - New `GET /api/tool-calls/:runId/:sequence` returning the four lazy fields. 404 when not found, 400 when sequence is malformed.

## Frontend changes
- `src/ui/src/api/types.ts` — remove the four lazy fields from `RunToolCall`; add `ToolCallDetail`.
- `src/ui/src/api/client.ts` — add `fetchToolCallDetail(runId, sequence)`.
- `src/ui/src/api/queries.ts` — add `useToolCallDetail(runId, sequence, enabled)` with `staleTime: Infinity`.
- `src/ui/src/pages/TicketDetailPage.tsx` (`LogRow`) — when the expanded row is a tool call, call the hook gated on `open`, show a spinner while pending, then render thought/input/output. Cache reuses on close+reopen.

## Event payload lazy-load (parallel scope)
Applied the same lazy-load pattern to `TicketEvent.payloadJson`, which is also heavy and not needed until a user expands an event row:
- `src/db/client.ts` — strip `payloadJson` from `TicketDetail.events`; add `getEventPayload(eventId)` returning `payload_json` (or `null`).
- `src/manager/routes.ts` — new `GET /api/events/:id/payload` returning `{ payloadJson }`. Returns 200 + `{ payloadJson: null }` for both 'event not found' and 'event has no payload' (no caller needs to distinguish today).
- `src/ui/src/api/types.ts` — remove `payloadJson` from `TicketEvent`; add `EventPayload`.
- `src/ui/src/api/client.ts` — add `fetchEventPayload(eventId)`.
- `src/ui/src/api/queries.ts` — add `useEventPayload(eventId, enabled)` with `staleTime: Infinity`.
- `src/ui/src/pages/TicketDetailPage.tsx` (`LogRow`) — for event rows, fetch payload on expand. No spinner: `source`/`type`/`summary` stay visible and the payload renders silently when it arrives.
