# DEN-2576 — bear metal healthcheck

## Problem

The bear-metal container healthcheck (`GET /api/health`) was unreachable during
startup whenever a worker environment builder was configured. The builder can
take up to 30 minutes (`TIMEOUT_MS = 30 * 60 * 1000`), and the previous startup
order called `app.listen` only after `runWorkerEnvironmentBuilder` and
`db.initSchema` finished. Docker/k8s healthchecks therefore failed for the entire
bootstrap window.

## Change

- Split `src/manager/app.ts` into:
  - `createBootstrapApp()` — express app with only `/api/health` mounted.
  - `mountFullApi(app, db, maxIterations)` — adds DB-backed `/api`, static UI, SPA fallback.
  - `createApp(...)` kept as a convenience that composes both.
- `src/manager/index.ts` creates the bootstrap app and calls `app.listen` *before*
  `runWorkerEnvironmentBuilder` and DB init, then mounts the full API once both
  finish. Test-ticket mode skips the server (preserves previous behavior).
  Shutdown handler tolerates the no-server case. Explicit guard on `app`/`server`
  before mounting the full API so the invariant is type-checked rather than
  relying on the implicit `process.exit` in the test-ticket branch.
- Removed the duplicate `/health` route from `createRouter` (it lived under `/api`
  already and is now owned by the bootstrap app).

## Files

- `bear-metal/src/manager/app.ts`
- `bear-metal/src/manager/index.ts`
- `bear-metal/src/manager/routes.ts` (removed duplicate `/health`)

## Verification

- `npx tsc --noEmit` — clean.
- `npx vitest run` — passes (pre-existing flakes in `worker/pi.test.ts` and
  `worker/task-worker.test.ts` are unrelated).
- Manual: `curl localhost:<port>/api/health` returns 200 immediately after process
  start, before the env builder completes.
