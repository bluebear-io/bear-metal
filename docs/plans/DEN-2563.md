# DEN-2563 — Reconcile stale `waiting_for_human` tickets

## Problem
`ticket_statuses.status = 'waiting_for_human'` rows can become stale because
the scheduler only refreshes tracked (non-released) latest task rows. When a
slot is released or the latest task row no longer goes through the tracked
refresh path, terminal Linear/PR state never gets reconciled and the UI keeps
showing stale `waiting_for_human`.

## Approach
1. New DB method `listStaleWaitingForHuman()` returns the latest task row per
   ticket where `ticket_statuses.status = 'waiting_for_human'` AND
   `tasks.slot_status = 'released'`. Returns enough metadata
   (`input_json`, `result_json`) to recover known PR refs without going back
   to Linear attachments.
2. New DB method `listWaitingForHumanTicketIds()` so admission can exclude
   tickets that are already in `waiting_for_human` (covers released rows that
   are still delegated to Bear Metal, preventing accidental re-admission as a
   "new" task).
3. New scheduler step `reconcileStaleWaitingForHuman()` runs each tick after
   the tracked refresh:
   - Fetch the live Linear ticket.
   - Linear terminal (`completed` / `canceled` / state name `Merged`) →
     `ticket_statuses.status = 'completed'`, ensure slot `released`,
     emit `ticket_completed` event. No hand back.
   - Else, query GitHub per known PR. If all known PRs are resolved
     (merged or closed) AND at least one was merged → hand back if still
     delegated to Bear Metal, `ticket_statuses.status = 'completed'`,
     ensure slot `released`, emit `ticket_completed` event.
   - Otherwise (multi-PR partial, closed-unmerged only, clean open) →
     leave the row alone. Do NOT release a non-terminal
     `waiting_for_human` row.
4. `admitNewTickets` extends its exclusion set with the
   `waiting_for_human` ticket ids so a released stale row is not silently
   re-admitted as a brand-new task.

## Files
- `bear-metal/src/db/client.ts` — add methods to `DbClient` and `SqlDbClient`.
- `bear-metal/src/manager/scheduler.ts` — add `reconcileStaleWaitingForHuman`,
  wire into `tick()`, plumb the new exclusion list through `admitNewTickets`.
- `bear-metal/src/manager/scheduler.test.ts` — add the six test cases listed
  in the ticket.
