# DEN-2373 — Change bear-metal startup log message to "hello! i'm up!"

## Goal
Replace the manager process's startup banner log message so bear-metal logs `hello! i'm up!` when it comes online.

## Scope
Single-line string change. No behavioral, schema, API, or infra changes.

## Files touched
- `src/manager/index.ts` — in the `app.listen` callback, replace
  `"🐻 Bear Metal is awake and hungry for tickets — let's ship some code!"` with
  `"hello! i'm up!"`. The adjacent `"health server listening"` log is left untouched
  (it describes the HTTP health endpoint, not the bear-metal process startup banner).

## Out of scope
- `src/backend/index.ts` dashboard backend "listening" message — that's a different
  service's listen-on-port log, not the bear-metal manager startup banner.
- Worker / dispatch / pi per-task lifecycle logs.

## Verification
- `grep` confirms the old banner string is removed and `"hello! i'm up!"` is present in
  `src/manager/index.ts`.
- No tests assert against this log string (verified by `grep`).
- Typecheck / build via existing CI.
